# 借阅模块 API 联调说明

**负责人：** 后端 B（借阅业务）  
**模块路径：** `/bookadmin/borrow`  
**本地服务地址：** `http://localhost:8080`（默认端口，以实际启动日志为准）

---

## 一、通用约定

### 1. 请求方式

| 项目 | 说明 |
|------|------|
| HTTP 方法 | 全部为 **POST**（不要用 GET，浏览器地址栏直接访问会返回 405） |
| 参数传递 | **Query 参数**（URL 后 `?key=value`），当前接口不使用 JSON Body |
| Content-Type | 默认即可，或 `application/x-www-form-urlencoded` |

### 2. 统一响应格式 `Result`

所有接口均返回 JSON，结构如下：

```json
{
  "code": 1,
  "msg": "success",
  "data": "具体业务信息或 null"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | Integer | **1 = 成功**，**0 = 失败** |
| `msg` | String | 成功时为 `"success"`，失败时为中文错误描述 |
| `data` | String / null | 成功时的业务提示；失败时一般为 `null` |

**前端建议判断逻辑：**

```javascript
if (response.code === 1) {
  // 成功，展示 response.data
} else {
  // 失败，展示 response.msg
}
```

## 二、接口列表

### 1. 借书

| 项目 | 内容 |
|------|------|
| URL | `POST /bookadmin/borrow/lend` |
| 示例 | `http://localhost:8080/bookadmin/borrow/lend?readerId=1&bookId=2` |

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| readerId | Integer | 是 | 读者 ID |
| bookId | Integer | 是 | 图书 ID |

**成功响应：**

```json
{
  "code": 1,
  "msg": "success",
  "data": "借阅登记成功！"
}
```

**失败响应示例：**

```json
{
  "code": 0,
  "msg": "读者不存在，借阅失败！",
  "data": null
}
```

**常见失败 `msg`：**

- `读者ID和图书ID不能为空！`
- `读者不存在，借阅失败！`
- `读者状态异常，无法借阅！`
- `该书已被借出，无法重复借阅！`
- `该读者类型未配置借阅规则，借阅失败！`
- `已达最大借阅数量，无法继续借阅！`

---

### 2. 还书

| 项目 | 内容 |
|------|------|
| URL | `POST /bookadmin/borrow/return` |
| 示例 | `http://localhost:8080/bookadmin/borrow/return?borrowId=1` |

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| borrowId | Integer | 是 | 借阅流水号（借书成功后由数据库生成） |

**成功响应：**

```json
{
  "code": 1,
  "msg": "success",
  "data": "归还成功！"
}
```

**失败响应示例：**

```json
{
  "code": 0,
  "msg": "这本书已经归还过了，请勿重复操作！",
  "data": null
}
```

**常见失败 `msg`：**

- `借阅流水号不能为空！`
- `系统中找不到这条借阅记录，操作失败！`
- `这本书已经归还过了，请勿重复操作！`

> 说明：若超期还书，后端会自动计算罚款并写入 `Fine` 表，前端展示「归还成功」即可。

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| borrowId | Integer | 是 | 借阅流水号 |

**成功响应：**

```json
{
  "code": 1,
  "msg": "success",
  "data": "续借成功！"
}
```

**常见失败 `msg`：**

- `借阅流水号不能为空！`

### 4. 逾期催还检查

| 项目 | 内容 |
|------|------|
| URL | `POST /bookadmin/borrow/notify` |
| 示例 | `http://localhost:8080/bookadmin/borrow/notify` |

```json

  "code": 1,
```

**失败响应：**

```json
{
  "code": 0,
  "msg": "系统异常，逾期催还检查执行失败！",
  "data": null
}
```

## 三、前端调用示例（Axios）

```javascript
const BASE = 'http://localhost:8080';

// 借书
axios.post(`${BASE}/bookadmin/borrow/lend`, null, {
  params: { readerId: 1, bookId: 2 }
}).then(res => {
  if (res.data.code === 1) alert(res.data.data);
  else alert(res.data.msg);
});

// 还书
axios.post(`${BASE}/bookadmin/borrow/return`, null, {
  params: { borrowId: 1 }
});

// 续借
axios.post(`${BASE}/bookadmin/borrow/renew`, null, {
  params: { borrowId: 1 }
});

// 逾期催还
axios.post(`${BASE}/bookadmin/borrow/notify`);
```

## 四、Postman 联调步骤

1. Method 选择 **POST**
2. URL 填入完整地址（含 Query 参数）
3. 点击 **Send**
4. 查看 Body 中 JSON 的 `code` 字段判断成败

**借书示例 URL：**

```text
http://localhost:8080/bookadmin/borrow/lend?readerId=1&bookId=2
```

## 五、联调前置条件

1. 后端已启动（终端出现 `Started LibrarySystemApplication`）
2. MySQL 已创建 `library` 库，并执行建表脚本（见 `src/main/resources/sql/schema.sql`）
3. 借书前 `readerId` 需在 `reader` 表中存在
4. `Rule` 表中需有与读者 `reader_type` 匹配的借阅规则
5. `borrowId` 来自借阅记录（当前借书接口不返回该 ID，需从列表页或数据库查询；后续可扩展）

## 七、常见问题

**Q：浏览器打开接口 URL 报 405？**  
A：接口只支持 POST，请用 Postman 或 Axios 发 POST 请求。

**Q：HTTP 200 但业务失败？**  
A：看 JSON 里 `code` 是否为 0，错误信息在 `msg` 字段。

**Q：跨域报错？**  
A：前端与后端端口不同时，需后端配置 CORS，联调阶段请联系后端处理。
